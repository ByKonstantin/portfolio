# Изображения кейсов

Положите сюда файлы — они будут доступны по путям `/images/имя-файла`.

## Имена файлов (по cases.json)

- `case-1-preview.webp` — превью в блоке screens
- `case-1-01.webp`, `case-1-02.webp` — слайды галереи

---

## Форматы и оптимизация

### Рекомендуемый формат: **WebP**
- На 25–35% меньше PNG/JPEG при том же качестве
- Поддержка во всех современных браузерах (Chrome, Firefox, Safari, Edge)
- Качество 75–85 для фото, 90+ для скриншотов UI

### Альтернативы
- **AVIF** — ещё легче, но Safari поддерживает с версии 16
- **PNG** — если нужна прозрачность
- **JPEG** — для фото без прозрачности

### Размеры
- Превью/мокап: до **800px** по длинной стороне
- Галерея: до **1200–1600px**

### Оптимизация
Vite **не сжимает** изображения при сборке. Сжимайте заранее:
- [Squoosh](https://squoosh.app/) — WebP, AVIF, настройка качества
- [TinyPNG](https://tinypng.com/) — PNG, JPEG
